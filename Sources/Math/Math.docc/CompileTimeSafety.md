# Compile-Time Dimension Type Safety

Learn how to enforce dimension constraints at compile time using unit generic parameters.

## Overview

Historically, physical dimensions were evaluated strictly at runtime in the `Math` library. To enable compile-time guarantees, the library now features the ``DimensionProtocol`` and the ``MathDimension`` type namespace.

### The Problem

When writing generic APIs that accept quantities, you often want to ensure that a quantity represents a specific dimension (e.g., length, not time) before running your function:

```swift
import CoreLocation
import Math

protocol PlaceService {
    func fetchNearbyPlaces<U: MathUnit>(
        coordinate: CLLocationCoordinate2D,
        radius: Math.Quantity<U>,
        completion: @escaping (Result<[Place], Error>) -> Void
    ) where U.Dimension == MathDimension.length // Compile-time check!
}
```

By parameterizing ``NamedUnit`` with a compile-time dimension type (e.g., `NamedUnit<MathDimension.length>`), Swift's compiler can enforce that you only pass length units into this method.

### How to Use It

All standard units generated by the `@PrefixedUnits` macro or declared in ``Units`` are parameterized by their respective dimension type.

#### Passing Correct Units
```swift
// Compiles perfectly!
let radius = Quantity(value: 500, unit: Units.meter)
placeService.fetchNearbyPlaces(coordinate: coords, radius: radius) { ... }
```

#### Prevent Mismatched Units
```swift
// Mismatched dimension (time vs length) - Compiler Error!
let timeLimit = Quantity(value: 30, unit: Units.minute)
placeService.fetchNearbyPlaces(coordinate: coords, radius: timeLimit) { ... }
```
